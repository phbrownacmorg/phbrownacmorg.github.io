    Body tags

    Structural elements

    <h1>, <h2>, <h3>, <h4>, <h5>, <h6>
    Headings
    <div>
    Marks off a block, but doesn't change the look by itself. Useful mainly for hanging style-sheet elements on.
    <span>
    Marks a phrase, but doesn't change the look by itself. Useful mainly for hanging style-sheet elements on.
    <address>
    Used for contact information. Sometimes it changes the look, sometimes it doesn't; depends on the browser.

    Links

    <a> (with href attribute)
    Hyperlink. href specifies a URL to link to.
    <a> (with id attribute)
    Hyperlink target. Hyperlink URL's that end in #targetname are pointing to an element of the form <a id="targetname">.

    Text tags

    <p>
    Encloses a paragraph.
    <br />
    Forces a
    line break.
    <pre>
    Encloses pre-formatted text. 
    This text is usually rendered in some fixed-width font (commonly Courier). White space is significant within this tag.
    <blockquote>
    Used to set off a long quotation.

    Some people use <blockquote> to indent text. Don't.

    <em>
    Encloses emphasized text. Usually rendered in italics, but this is not guaranteed.
    <strong>
    Encloses strongly emphasized text. Usually rendered in bold, but this is not guaranteed.
    <code>
    Encloses computer code. Whitespace may or may not be important; IE and Firefox do this differently.
    <var>
    Variable name. Often in code, but not always.
    <sub>
    Subscript.
    <sup>
    Superscript.
    <font>
    Specifies the font of the enclosed text directly. You'll see lots of these in code generated by (e.g.) Microsoft Word. Don't use it.

    Lists

    <ul>
    Unordered (bulleted) list. List elements are enclosed in <li></li>.
    <ol>
    Ordered (numbered) list. List elements are enclosed in <li></li>.
    <dl>
    Definition list. Terms to be defined are enclosed in <dt></dt>. Definitions are enclosed in <dd></dd>. There must be exactly one <dd> for each <dt>.

    Tables

    <table>
    Defines a table.
    <tr>
    Defines a single row of a table.
    <th>
    Defines a table header cell. Typically, the text in a <th cell is centered and bold. You can make a cell span multiple columns with the colspan attribute, or span multiple rows with the rowspan attribute.
    <td>
    Defines a table data cell. Typically, the text in a <td> cell is left-justified and normal weight. You can make a <td> cell span multiple rows and columns the same way you do with a <th> cell.

    Row and column groupings

    <tbody>
    Contains the main body of table cells. Must contain at least one row. Can be omitted if you don't have <thead> or <tfoot>.
    <thead>
    Defines some row(s) as header row(s). These can be scrolled independently of the rows in the <tbody>, and what rows are in the <thead> may also affect the drawing of lines between the cells. <thead> must be specified before <tbody>.
    <tfoot>
    Defines some row(s) as footer row(s). These can be scrolled independently of the rows in the <tbody>, and what rows are in the <tfoot> may also affect the drawing of lines between the cells. <tfoot> must be specified before <tbody>, even though the <tfoot> row(s) are displayed at the bottom of the table.
    <col>
    Defines a single column.
    <colgroup>
    Defines a group of columns. Useful for presetting column widths, or for controlling where the vertical lines go. Note that <colgroup> and <col> both happen before the rows are defined; the data cells are always defined inside the rows, not in <colgroup> or <col>.

    Images

    <img>
    Inserts an image. Important attributes:
    src
    URL where the image may be found.
    alt
    Text to display if the image can't be displayed. (Why is this important?)
    width
    Width of the image. If this attribute is given, the image will be scaled to this width.
    height
    Height of the image. If this attribute is given, the image will be scaled to this height.
    usemap
    Specifies the URL of a client-side image map to use.
    <map>
    Defines a client-side image map, so that clicking on the image will do something. Important attribute:
    name
    Gives a name to the image map, so a URL can find it (same way as with an <a name="target"> element).
    <area>
    Defines an area within an image map, and where to go if the user clicks there. Important attributes:
    shape
    One of rect (defines a rectangular region), circle q(defines a circular region), poly (defines a polygonal region), or default (this area takes up the entire map).
    coords
    Coordinates defining the region, relative to the top left corner of the <map>. Which coordinates they are depends on the shape attribute, as follows:
    • rect: left X, top Y, right X, bottom Y
    • circle: center X, center Y, radius
    • poly: series of x,y coordinate pairs defining vertices in order along the outside of the polygon
    href
    Where to go if the user clicks within the area.
    nohref
    Indicates that this area is inactive. Normally used to cut an inactive chunk out of a larger active area.
    alt
    Text to indicate what's there if the image cannot be displayed.

    Horizontal rules

    <hr />
    Makes a horizontal line across the page. It has several attributes for controlling appearance, all of which are deprecated in favor of style sheets.
